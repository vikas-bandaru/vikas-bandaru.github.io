JavaScript Events Tutorial

This webpage demonstrates how to use JavaScript events to make your webpages interactive.

Event

An event is an action or occurrence that takes place in the browser, such as a user clicking a button, typing into a text input, or the page finishing loading. Events are used to trigger the code that responds to these actions and makes changes to the webpage.

Examples

  • Keyboard Events: These events occur when a user interacts with the keyboard, such as pressing a key or releasing a key. Eg. keyup, keydown, etc.
  • Mouse events: These events occur when a user interacts with the mouse, such as clicking a button, scrolling, or dragging. Eg. click, mouseover, mouseout, etc.
  • Form events: These events occur when a user interacts with a form element, such as submitting a form or resetting a form. Eg. submit, change, etc.
  • Load Event: Occurs when a page or an image has finished loading. Eg. load
  • Media events: These events occur when a media element, such as a video or audio player, is played, paused, or stopped.

Event Handler

An event handler is a block of code that runs in response to a specific event. It is responsible for defining the behavior of an element when an event is triggered. For example, an onclick event handler can be used to execute a JavaScript function when a button is clicked.

Event Listener

An event listener acts as a middle-person between an Event and it's associated Event Handler. It is a function that waits for a specific event to occur on an HTML element and then executes the associated code. Event listeners are not specific to a single element or event, and can be attached to multiple elements or events. Event listeners are typically added using the addEventListener() method, which typically takes two parameters: the type of event to listen for, the function to be executed when the event occurs. Unlike inline event handling, event listeners are defined separately from the HTML code.

Types of Event Handlers

  1. Inline event handlers: This involves adding the event handler function directly to the HTML element, using the on prefix and the name of the event. For example:

    <button onclick="alert('Button clicked!')">Click me</button>

    However, inline event handling is not always the best approach, as it can clutter the HTML code and make it harder to read and maintain.

  2. DOM event handlers: This involves adding the event handler function to the DOM element using Event Listeners.

    This method is considered to be more flexible and maintainable than inline event handling.

    Beginner-friendly Code:
     

    <button id="btn1">Click me</button>
    <script>
      const button = document.getElementById('btn1');
      button.addEventListener('click', buttonClicked);
      function buttonClicked() {
        alert('Button clicked!');
      }
    </script>

    The left side code can also be written as the following and is also used often by advanced developers:

    <button id="btn1">Click me</button>
    <script>
      const button = document.getElementById('btn1');
      button.addEventListener('click', function() {
          alert('Button clicked!');
        }
      );
    </script>

Example 1: OnClick Event

The onclick attribute is an example of an inline event handler. Click the button to change the text in the Output box.

Code:

<button onclick="changeText()">Click me</button>
<p id="example1">This is some text.</p>
<script>
function changeText() {
  var para = document.getElementById("example1");
  para.innerHTML = "<em>This is italicised text.</em>";
}
</script>

Output:

This is some text.

Explanation:

  1. <button onclick="changeText()">Click me</button>

    This line creates a button element with the text "Click me" and attaches an event listener (onclick) to it. Specifically, it listens for a click event and when the button is clicked, it will call the function changeText().

  2. <p id="example1">This is some text.</p>

    This line creates a paragraph element with the text "This is some text." and an id of "example1". The id attribute is used to uniquely identify this element so it can be manipulated with JavaScript.

  3. function changeText() { }

    This block of JavaScript code defines the function changeText() which is called when the "Click me" button is clicked.

  4. var para = document.getElementById("example1");

    The getElementById() method is a built-in method of the document object in JavaScript. It allows you to select an element in the HTML document based on its unique ID attribute value. In this case, the ID "example1" belongs to the p element that is to be manipulated using JavaScript. The getElementById() method is used to retrieve a reference to the p element, and it is stored in the variable para for further manipulation.

  5. para.innerHTML = "<em>This is italicised text.</em>";

    It is modifying the content of the HTML element with the ID of "example1" that was previously assigned to the variable p. The .innerHTML property of an HTML element allows you to get or set the HTML content inside an element. In this case, the line is setting the innerHTML property of the p element to a new string that contains an HTML em element with the text "This is italicised text." As a result of this line of code, the text "This is some text." that was previously inside the p element will be replaced with the new string containing the em element, which will display the text in italics.

Example 2: OnMouseOver Event

The onmouseover attribute is also an example of an inline event handler. Hover over the text to change the background color:

This is some text.

Code:

<p id="example2" onmouseover="changeColor()">This is some text.</p>
<script>
function changeColor() {
  var para = document.getElementById("example2");
  para.style.backgroundColor = "yellow";
}
</script>

Output:

This is some text.

Explanation:

  1. <p id="example2" onmouseover="changeColor()">This is some text.</p>

    This line creates a p element with the text "This is some text." and attaches an event listener (onmouseover) to it. Specifically, it listens for a "on mouse over" event and when the mouse is moved over the p element's text, it will call the function changeColor(). The onmouseover event is a mouse event that triggers when the user moves the mouse pointer over the element. This event is set to call the changeColor() function, which can be defined in a separate <script> element or in an external JavaScript file.

  2. para.style.backgroundColor = "yellow";

    para is a reference to p element in the webpage, obtained using the document.getElementById() method. style is a property of the HTML element that provides access to its styling information. It is an object with properties that correspond to the various style properties (e.g. color, font-size, background-color) that can be applied to the element.

    backgroundColor is a property of the style object that sets or returns the background color of the element. Therefore, para.style.backgroundColor = "yellow"; sets the background color of the HTML element referred to by para to yellow.

Example 3: OnLoad Event

The text should change when the page loads:

Code:

<p id="example3"></p>
<script>
document.addEventListener("DOMContentLoaded", function() {
  document.getElementById("example3").innerHTML = "This text was added using JavaScript on page load!";
});
</script>

Output:

Explanation:

  1. document.addEventListener("DOMContentLoaded", function() {  //event handling code  });

    This code adds an event listener to the document that listens for the "DOMContentLoaded" event, which fires when the HTML document has finished loading and parsing. When the event is detected, the function specified as the second argument is called.

  2. The function gets the element with the ID "example3" using the document.getElementById() method, and sets its innerHTML property to the specified string. This effectively adds the text "This text was added using JavaScript on page load!" to the HTML document.

Example 4: OnSubmit Event

Enter your name and click the "Submit" button:

Code:

<form onsubmit="return greetUser()">
  <div class="form-group">
    <label for="name">Name:</label>
    <input type="text" class="form-control" id="name">
  </div>
  <button type="submit" class="btn btn-primary">Submit</button>
</form>
<p id="example4"></p>
<script>
function greetUser() {
  const name = document.getElementById("name").value;
  document.getElementById("example4").innerHTML = "Hello, " + name + "!";
  return false;
}
</script>

Output:

Explanation:

    This code demonstrates a basic form that accepts user input and greets the user by name. The form includes an input field for the user's name, a submit button, and a paragraph element where the greeting will be displayed.

  1. <form onsubmit="return greetUser()">

    The onsubmit attribute is used to attach an event handler to the form that will be executed when the user submits the form. In this case, the event handler is the greetUser() function.

  2. The greetUser() function first retrieves the user's name from the input field with the ID "name" using the document.getElementById() method. It then uses string concatenation to create a personalized greeting message and sets the innerHTML property of the paragraph element with the ID "example4" to display the greeting message.
  3. onsubmit="return greetUser()" and return false; statement:

    The return false statement is used to prevent the form from submitting and reloading the page, which is the default behavior when a form is submitted. This allows the greeting to be displayed without the page refreshing.
